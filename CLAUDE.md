## Project Overview

BookTrader is a cross-platform Expo-based React Native app for local peer-to-peer book trading in Prestige Shantiniketan, Whitefield (Bengaluru), with plans to expand across India. It's a single-screen monolithic app with all logic in App.tsx using local React state and Firestore for persistence.

## Build and Development Commands

```bash
# Install dependencies
npm install

# Start the app (opens Expo CLI menu for iOS/Android/Web)
npm start

# Start on specific platforms
npm run android          # Android
npm run ios            # iOS
npm run web            # Web

# Type checking
npm run typecheck      # Run TypeScript compiler

# Android native build
npm run prebuild:android    # Generate Android project from Expo config
```

Android builds additionally require the android/ directory generated by prebuild. GitHub Actions builds debug APKs via `.github/workflows/android-apk.yml` on push to main/dev/feature/* branches, PRs, or manual dispatch.

## Project Structure

- **App.tsx** (62KB) ‚Äî Single monolithic component containing all screens, state management, and business logic. React hooks manage auth state, user profile, listings, offers, and modals.
- **index.ts** ‚Äî Expo entry point that registers App as root component.
- **src/config/firebase.ts** ‚Äî Firebase/Firestore initialization using environment variables; exports null objects in demo mode.
- **src/services/bookAi.ts** ‚Äî Google Books API integration for auto-filling book metadata (title, author, cover image) from a title or author query.
- **src/types.ts** ‚Äî TypeScript interfaces: `UserProfile`, `BookListing`, `BookDraft`, `TradeOffer`, `TradeMessage`, `Coordinates`, and enums for `ListingStatus` and `OfferStatus`.
- **src/data/demo.ts** ‚Äî Demo data: `demoUser`, `demoListings`, `demoOffers` used in offline mode when Firebase config is missing.
- **src/theme.ts** ‚Äî Design tokens: `colors`, `radii`, `spacing`.
- **firebase/** ‚Äî Firestore and Storage security rules (rules_version 2).
- **app.json** ‚Äî Expo config with app permissions, platform-specific settings, and plugin configuration (image-picker, location, notifications).
- **eas.json** ‚Äî EAS Build configuration for preview and production builds.

## Architecture & State Management

**Single Component Monolith:** All UI and logic resides in the App component using React hooks (`useState`, `useEffect`, `useMemo`). No Redux, Context, or other state library.

**State Shape:**
- `firebaseUser` ‚Äî Firebase Auth user object or null
- `authLoading` ‚Äî Boolean flag during auth initialization
- `profile` ‚Äî Current user's profile from Firestore (UserProfile)
- `demoProfile` ‚Äî In-memory profile used in demo mode
- `demoMode` ‚Äî Boolean; true if Firebase config is missing
- `listings` ‚Äî Array of BookListing docs from Firestore
- `offers` ‚Äî Array of TradeOffer docs from user is involved in
- `activeTab` ‚Äî Current screen tab: 'market' | 'add' | 'trades' | 'profile'
- `busy` ‚Äî Loading flag for async operations
- `search` ‚Äî Marketplace search text filter

**Real-time Sync:** Uses Firestore `onSnapshot()` listeners to subscribe to listings and offers collections; filters client-side to show only active listings (not expired) and offers where user is requester or listing owner.

**Demo Mode:** When `.env` Firebase values are missing, the app runs in demo mode with in-memory data (`demoListings`, `demoOffers`, `demoUser`) and no persistence.

## Authentication & User Onboarding

- **Auth Providers:** Firebase Auth with Google (via Expo Auth Session) and email/password.
- **Required Fields:** Legal name, city, community, optional wishlist.
- **Location:** Requested via `expo-location` for distance calculations; optional.
- **Push Tokens:** Registered with Expo Notifications if permissions granted; stored in Firestore.

## Core Features

1. **Marketplace** ‚Äî Browse nearby book listings, sorted by distance. Search by title/author/ISBN. Suggested matches from wishlist.
2. **Add Listing** ‚Äî Book metadata auto-filled from Google Books API; fields: title, author, edition, description, wants. Listings expire after 14 days.
3. **Trade Offers** ‚Äî Request a trade (pending ‚Üí accepted/declined ‚Üí completed). Chat after acceptance.
4. **Ratings** ‚Äî Post-trade ratings (immutable once created).
5. **Profile** ‚Äî View/edit profile, manage listings, view ratings.

## Firestore Collections & Rules

**Collections:**
- `users/{userId}` ‚Äî User profile, wishlist, ratings, push token
- `listings/{listingId}` ‚Äî Book listing with owner info, images, wants, status
- `tradeOffers/{offerId}` ‚Äî Trade request with messages array, status
- `ratings/{ratingId}` ‚Äî Immutable rating (create-only)

**Key Rules:**
- Users can only update their own profile.
- Listing owners can update; claimedBy users can also update.
- Trade offers readable by both participants; both can update.
- Storage allows authenticated users to write to `/{userId}/listings/` directory.

## Key Patterns & Utilities

- **Distance Calculation** ‚Äî `kmBetween(a, b)` uses Haversine formula; formats as "Xm" or "Xkm" via `formatDistance()`.
- **ID Generation** ‚Äî `makeId(prefix)` creates `prefix-timestamp-randomstring`.
- **Status Coloring** ‚Äî `statusColor(status)` maps to theme colors (green=open/accepted, amber=pending/claimed, red=declined, gray=completed).
- **Data Mapping** ‚Äî `mapListing()` and `mapOffer()` cast Firestore data to typed objects with defaults.

## UI Screens (in App.tsx)

App renders a tab-based UI with screens for:
- **Market** ‚Äî Browse listings with search/filter
- **Add** ‚Äî Publish a new listing
- **Trades** ‚Äî View trade offers
- **Profile** ‚Äî User profile & settings

Authentication screens (login/register) render before tabs if user is not authenticated.

## Notes for Future Development

- **Monolith Risk:** App.tsx is 62KB. Breaking into multiple components or extracting state management (Context API, Zustand) would improve maintainability.
- **Testing:** No test files present. Consider adding unit tests (utilities like `kmBetween`, `makeId`) and integration tests (auth flows, Firestore operations).
- **Images:** Book cover images stored in Firebase Storage (optional). Missing images don't break the app.
- **Notifications:** Expo Notifications configured for local 14-day listing expiry reminders; server-side reminders require backend credentials.
- **New Architecture:** `newArchEnabled: true` in app.json; uses React Native's new JSI-based architecture.
